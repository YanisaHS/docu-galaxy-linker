What the findings mean

Broken doc refs

A link in one page points at a file path that doesn't exist on disk. Readers who click it land on a 404 (or the docs build fails). Usually caused by a rename or move where the linker wasn't updated.

Example: [install guide](how-to/install.md) when the file is now how-to/installation.md.

Broken anchors

A link to a heading on a page where that heading no longer exists. The reader lands on the page but doesn't scroll to anything meaningful.

Example: [](architecture.md#authentication) when the page no longer has an "Authentication" heading.

Broken labels

A {ref} or :ref: cross-reference using a label name that was never defined anywhere with (label-name)=. Sphinx fails to resolve it and the link renders as plain text.

Orphans

A document with zero incoming internal links — no other page in the docs links to it. The only way a reader reaches it is direct URL or search. Often a sign of a forgotten page, a page that should appear in nav, or a stale file left behind during restructuring.

Not always bad: a release-notes index page reached only via a top-level menu can be legitimately orphan-in-the-link-graph.

Dead ends

A document with zero outgoing internal links. Once a reader lands on this page, there is nothing to click — no "see also", no "next step", no link back to the parent overview. Bad for journey-building and discoverability, and the most common driver of bounces.

Common causes: stub pages, short reference cards that should link to related how-tos, pages where the author forgot a "Related" section.

Diataxis crosses

An internal link where the source and target sit in different Diataxis sections (tutorial → how-to, how-to → reference, etc.). Some crossing is healthy — a tutorial linking to a reference for syntax is fine. A high number, especially tutorial → reference, signals the tutorial may have drifted toward being a reference doc.

Diataxis purity

Of all internal links between documents in named Diataxis sections (excluding meta pages like index.md), the percentage that stay within their section. Higher is better: each section stays internally coherent. A sudden drop after a restructure is usually investigation-worthy.

Reach @ 3

Starting from the entry page (usually index.md), the percentage of documents reachable in three or fewer link hops. Approximates "how much of your docs can a reader discover from the front door?" Higher is better; low values signal weak top-level navigation.
